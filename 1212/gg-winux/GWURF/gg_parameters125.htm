RECOVERYOPTIONS

Valid For

Extract

Description

Use the RECOVERYOPTIONS parameter to control how Extract handles the re-writing of data to the trail after it fails while in the process of writing transaction data.

Append Mode

By default, Extract operates in append mode. In this mode, Extract writes a recovery marker to the trail after a failure, and then it appends the recaptured recovery data to the file, instead of overwriting the original data. Append mode retains a history of all prior data.

During recovery in append mode, Extract determines the identity of the last complete transaction that it wrote to the trail and then completes its recovery when the commit record for that transaction is encountered in the data source. A data pump or Replicat starts reading again from that recovery point. Extract starts capturing new data at the start of the next committed transaction after the recovery point, and this new data is appended to the trail file.

Overwrite Mode

Overwrite mode is another version of Extract recovery that was used in versions of Oracle GoldenGate prior to version 10.0. In these versions, Extract overwrites the original transaction data with recaptured data, instead of appending the new data. The next transaction that Extract captures is the first one after the last read checkpoint in the data source.

In overwrite mode, there is a probability that the overwrite might not deposit precisely the same record images, in precisely the same sequence, as the originals that are being overwritten. This can happen when Extract must fetch large objects or when Extract configuration parameters have been changed. Between the time when overwrite activity begins and the time when Extract reaches the end of the trail file, any such changes will create a misalignment at the leading edge of the overwritten portion of the file. When this happens, the end of the last record that was rewritten does not fall on a boundary that is shared by the beginning of a record that was written by the previous instance of Extract. A Replicat or data pump that is trying to read from one trail record to the next starts somewhere in the middle of a corrupted record and then abends with an error.

Recommendations

Do not change RECOVERYOPTIONS from the default of APPENDMODE unless instructed to do so by an Oracle Support analyst. If Extract is permitted to overwrite existing data in the trail, it might be harder for Oracle GoldenGate to recover after a failure, and it might cause the loss of data.

In some cases, Extract automatically reverts to overwrite mode to support backward compatibility if the version of Oracle GoldenGate on the target is older than Oracle GoldenGate version 10. Older versions do not support append mode.

Parameter Dependencies

There is a dependency between the RECOVERYOPTIONS parameter and the FORMAT option of EXTFILE, EXTTRAIL, RMTFILE, and RMTTRAIL. When RECOVERYOPTIONS is set to APPENDMODE, the FORMAT option must be set to RELEASE 10.0 or greater. When RECOVERYOPTIONS is set to OVERWRITEMODE, the FORMAT option must be set to RELEASE 9.5 or less.

Default

APPENDMODE

Syntax

RECOVERYOPTIONS {APPENDMODE | OVERWRITEMODE}
APPENDMODE

Appends new data to the original data in a trail file. This is the default.

OVERWRITEMODE

Overwrites original data with new data, starting at the most recent checkpoint position.

Example

RECOVERYOPTIONS OVERWRITEMODE
