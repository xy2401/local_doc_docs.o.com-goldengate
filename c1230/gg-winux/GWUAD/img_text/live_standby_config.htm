This graphic shows a live standby configuration. The active Oracle GoldenGate capture activity is on the active, or primary, source system. The active Extract writes to a local trail. A local data pump reads this trail and sends the data to a remote trail on the passive standby system. There, an active Replicat applies the data to the standby database. A passive Extract captures the changes through a local trail and data pump on that system, which sends the data to a remote trail on the active source for posting by a Replicat there.
